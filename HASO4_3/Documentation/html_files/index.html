Imagine Optic WaveKit - Labview API

Introduction

Welcome to the Labview API functions documentations!
To help you start with the Labview API, we organized the documentation in modules, each module corresponding either to a step of an optical metrology or an adaptative optic application, or to a group of shared basics concepts (see Modules page).
Each Module relies on one or several Labview classes, listed in its dedicated page (click on the Modules page, then on the module's name to access the page).
Each classe page contains a list of .vi files corresponding to the available classe's functions. (click on the Classes pages, then on the classe's name to access the page).

Important information

Installation The WaveKit installer for LabView installs all components in National instruments LabView 2018 folder : <ProgramFiles> Instruments 2018\user.lib. Event if your LabVIew is more recent, the wavekit is always installed into this folder. If you want to use a later LabView version, please copy the WaveKit.lib folder to the current LabView version user.lib folder. This labview API can be used in 32 or 64 bits environment. The dlls used by this API have been automatically loaded from the dlls (in 32 bits version) or dlls_x64 (in the 64 bits version)

Warning
Labview may display a warning of type "library loaded from new path" when opening examples. Just ignore this warning.
2D arrays (like phase values arrays or images) start index is upper-left corner (C convention), Visualization has to be transposed to fit the Labview natural orientation.
The Labview API may expose more functions that the ones appearing in this documentation. The undocumented functions correspond to higher-levels features, which are basically built using the documented ones.
