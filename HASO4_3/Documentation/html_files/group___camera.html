Camera device management
Interfacing the Camera

Detailed Description

Starter.

The Camera handles the physical image acquisition device (Haso sensor or Camera).
List of parameters available on all supported cameras:

  • "exposure_duration_us" (E_TYPE_INT)
  • "real_exposure_duration_us" (E_TYPE_INT)
  • "trigger_mode" (E_TYPE_STRING) with 4 values : "cam_internal","software","rising_edge", "falling_edge"

  • "user_timeout" (E_TYPE_INT)

    Please refer to the Camera specifications module for accessing specific camera parameters.

Configuring the Camera acquisition:

  • Camera has two operating (or synchronization) modes : sequential (synchronous) image acquisition or parallelized (asynchronous) image acquisition See E_CAMERA_SYNCHRO_MODE_T for details.
  • Camera has two acquisition modes : when GetImage is called, either the resulting image is a copy of what is in memory, or the result of an acquisition triggered by the method call. See E_CAMERA_ACQUISITION_MODE_T for details.
Warning
  • In E_CAMERA_ASYNCHRONOUS synchronization mode, Image can be retrieved only when ready Camera_AsyncImageReady returns true.
  • New image acquisition mode is not supported when camera runs in asynchronous mode.

enum E_CAMERA_ACQUISITION_MODE_T

Camera Acquisition modes

Acquisition start time can be controlled using the acquisition_mode.

  • In E_CAMERA_ACQ_LAST mode, a trigger signal is sent regularly and images are stored in a circular buffer.
    When user requests an image,
    • if the last image stored in circular buffer has already been sent once or if circular buffer is empty, user must wait until a new image is ready.
    • else the last image stored in circular buffer is sent.
  • In E_CAMERA_ACQ_NEW mode, images are integrated on demand. In this mode, integration start time is fully controlled, the counterpart being a possibly longer delay before image is available.
Enumerator
E_CAMERA_ACQ_LAST 

Camera returns the last image captured when Imop_Camera_GetImage is called.

E_CAMERA_ACQ_NEW 

Camera starts integrating a new image when Imop_Camera_GetImage is called.

enum E_CAMERA_SYNCHRO_MODE_T

Camera Synchronization modes

Camera has two operating modes.

  • In E_CAMERA_SYNCHRONOUS mode, each call to an image getter is blocking until image is ready.
  • In E_CAMERA_ASYNCHRONOUS mode, images are prepared in a separate thread when Camera_Start is called.
