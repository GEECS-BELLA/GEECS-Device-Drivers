Zaber A Series Instrument Driver Readme

To rate this instrument driver, go to the driver download page on the Instrument Driver Network.

1. Overview

Instrument Driver Technology: LabVIEW Plug and Play (project-style)
Manufacturer: Zaber Technologies
Supported Language(s): LabVIEW
Supported Model(s): A-BAR, A-BLQ, A-LAR, A-LSQ, A-LST, A-LAR, A-MCA, A-MCB2 (A-Series), X-BLQ, X-DMQ, X-FCR, X-FWR, X-GLP, X-GSM, X-GSR, X-JOY3, X-LDA X-LDQ, X-LDM, X-LHM, X-LRM, X-LRQ, X-LSM, X-LSQ, X-MCB1, X-MCB2, X-MCC1, X-MCC2, X-MCC3, X-MCC4, X-NMS, X-RSB, X-RST, X-RSW, X-VSR (X-Series)
Model(s) Tested: A-LSQ, X-MCB2, X-JOY3, X-RST, X-LDM, X-LDQ
Interface(s): RS232 Serial, TCP/IP
Firmware Revision(s) Tested: 6.18 (A-LSQ), 6.24 (X-RST), 6.25 (X-JOY3), 6.29 (X-MCB2), 7.03 (X-LDM, X-LDQ), 7.12 (X-MCC3)

Certified: Yes
NI Supported: Yes
Source Code Available: Yes

Driver Revision: 1.6.0
Original Release Date: 08/29/2014
Current Revision Date: 10/15/2021
Current Revision Release Date: 10/22/2021

2. Required Software

Some software components need to be installed before using this instrument driver. The minimum versions of these components are listed below, and can be downloaded from the Download Site.

VISA 5.0 or later

Refer to the LabVIEW Help for more information about software requirements. You access the LabVIEW Help by selecting Help»Search the LabVIEW Help.

The Zaber A-Series Control Panel.vi example requires a CSV file which is not included. An example CSV file and a Python script for updating it are available for download at http://www.zaber.com/wiki/Software/ASCII_Labview_Driver The CSV file and Python script are only needed by the example, and are not necessary to use the instrument driver.

3. Known Issues

To report issues or provide feedback about this instrument driver, please send an email to contact@zaber.com.

4. Revision History

The latest version of this and other LabVIEW and LabWindows/CVI instrument drivers can be downloaded at the Instrument Driver Network.

REV 1.6.0, 10/15/2021
Updated by: Satnam Singh Brar and Soleil Lapierre, Vancouver, BC.
  • Added a new Initialize TCPIP VI that allows easy connection to Zaber devices that are accessible over your network using TCP/IP. This includes new Zaber X-MCC Series controllers that have an Ethernet connector, or existing RS-232 devices connected to the network using Ethernet-to-serial bridge hardware or software.
  • Added a new Format TCPIP VISA Resource Name utility VI to assist in creating a TCP/IP connection string from a host name or IP address and a port number.
  • The following simple examples have been added to show usage of the new TCP/IP VIs. They can be found in the "Example Finder"
    • Zaber A Series TCPIP Move
    • Zaber A Series TCPIP Commands and Responses
  • Some palettes have been reorganized to eliminate empty spaces.
  • Added descriptions to all controls and indicators in the example VIs.
  • Data limit enforcement has been removed from all controls that previously had it. The limits still apply to devices; consult the documentation for legal values.
  • Added valid range and default value documentation to all relevant controls that were missing it.
  • Corrected some spelling mistakes in VI documentation.

REV 1.5.0, 07/24/2020
Updated by: Soleil Lapierre, Vancouver, BC.
  • The private Read (Single Response) VI, which is used by most other VIs in this driver, has been updated to be able to handle device responses consisting of multiple packets, a possibility that was introduced to the Zaber ASCII Protocol with the launch of four-axis controllers.
  • Bug fix: The Response String Array output of the Send Command VI will now include the message type identifier character at the start of each string for info and alert messages. Reply messages will omit the '@' identifier as they are the default expectation.

REV 1.4.0, 05/28/2020
Updated by: Soleil Lapierre, Vancouver, BC.
  • Multiple changes to support Zaber's new X-MCC line of controllers and new Firmware version 7:
    • Added the Write Analog Output and Read Analog Output VIs, as some Zaber devices now have analog output ports. The I/O example program has been updated to demonstrate the use of these VIs.
    • The Enable Lockstep VI now supports having more than two axes in a lockstep group, where the device has more than two axes.
    • Added the Set Lockstep Tolerance VI to allow customization of twist tolerance on multi-axis systems.
    • The Open Stream VI is joined by Open Stream (FW7) VI, which takes strings for the axis inputs instead of integers in order to allow specification of lockstep groups as stream axes, and a new Open Stream (Polymorphic) VI, which selects from the other two.
    • Added Stream Line On Axis Subset, Stream Circle On Axis Subset and Stream Arc On Axis Subset VIs that allow execution of these stream path segments on subsets of a stream's configured axes.

REV 1.3.0, 03/02/2020
Updated by: Soleil Lapierre, Vancouver, BC.
  • The Set Speed and Acceleration VI has been made obsolete due to mismatched unit conversions that cannot be fixed without a breaking change. Please use the new, separate Set Speed and Set Acceleration VIs instead.
  • The Convert Acceleration Units to Device Units and the Convert Device Units to Acceleration Units VIs have been made obsolete in order to make the list of acceleration units of measure correspond to those of the speed and position unit conversion VIs. Please use the new Convert Acceleration to Device Units and Convert Device Units to Acceleration VIs instead. The new ones do not support units of microsteps per second squared.
  • Fixed possible infinite loops in the Poll Until Idle VI and the private Read (Multiple Responses) VI if errors occur before they execute.

REV 1.2.2, 11/21/2019
Updated by: Soleil Lapierre, Vancouver, BC.
  • Fixed bugs in the Convert Acceleration Units to Device Units and the Convert Device Units to Acceleration Units VIs that were producing incorrect conversions of acceleration quantities.

REV 1.2.1, 03/01/2019
Updated by: Mike McDonald, Vancouver, BC.
  • Fixed a bug in the private Read (Single Response) VI that was sometimes causing extra whitespace to appear in one of the outputs.

REV 1.2, 10/05/2018
Updated by: Mike McDonald and Soleil Lapierre, Vancouver, BC.
  • Various VIs have been updated to use 64-bit integers to represent device positions, velocities and accelerations. In all cases this change should be backwards compatible, but using 64 bits where optional is recommended for users of high resolution devices such as direct drive linear stages.
    • Added new versions of the following VIs with 64-bit inputs or outputs for compatibility with Zaber firmware version 7 or later. In the cases where inputs were changed to 64 bits, the existing VI remains 32-bit and a new polymorphic version of the VIs has been added. Using the polymorphic version is recommended in new programs for future compatibility. In the cases where only new outputs were added, the new outputs are on previously unoccupied pins.
      • Lockstep Move
      • Lockstep Move at Velocity
      • Set Virtual Axis Speed
      • Virtual Axis Move
      • Virtual Axis Move at Velocity
      • Get Virtual Axis Info
      • Set Stream Speed and Acceleration
      • Stream Line
      • Stream Circle
      • Stream Arc
      • All six unit conversion VIs under Utility / Low-Level.
    • The Read Current Position VI can now internally handle device positions outside the range of a 32-bit integer. The output type is still a double, which means that precision may be lost when dealing with very large numbers.
  • Updated the private Read (Single Response).vi to handle more possible errors, including:
    • Validating checksums if the comm.checksum setting is enabled.
    • Checking that the response starts with an expected Message Type (@, !, or #).
    • Checking that the Reply Flag is an expected value (OK or RJ).
    • Returning an empty Response Cluster if the response is an Alert or Info message.
  • Added the private Validate Checksum.vi.
  • Added a First Response Bundle output to the private Read (Multiple Responses).vi and Request.vi, and added a First Response Bundle output to Send Command.vi and First Response Data output to Get Setting.vi.
  • Updated all subVIs and examples to used these new outputs where applicable.
  • Fixed a bug in the circle part of the Streaming example; it was using the arc endpoint coordinates as the circle center by accident.
  • Changed the Get Device IDs VI to suppress expected errors when attempting to get the peripheral ID from devices that don't have one.
  • Corrected incorrect information in Device ID Lookup VI documentation about the location of the associated example CSV data file.
  • The format of the CSV file used by the Device ID Lookup VI has changed, and the online downloadable example updated accordingly. The VI now supports devices that do not have a resolution setting, such as direct drive stages, and will correctly report their microstep size.
